# AEM Screens简介 {#introduction}

**AEM(Adobe Experience Manager)Screens是一款数字标牌解决方案** ，它允许您创作、发布和播放涉及不同类型场所内显示屏的动态交互式数字体验，并制定全面的全渠道数字营销战略。

AEM Screens允许您创建：

* **专用数字菜单板**
* **产品推荐人**
* **背景生活方式影像**

此外，Screens还根据部署这些应用程序的域为客户和员工提供许多独特的应用程序，例如：

* **交互式显示**
* **寻路**
* **品牌**
* **为环境添加环境**

使用AEM Screens创建和管理数字标牌网络非常简单直观。 播放器应用程序托管由客户或实施合作伙伴为AEM Screens构建的内容渠道。 *位置* ，管理预定义的位置层次结构并包含显示。 每个 *显示屏* ，都附有显示不同设备和屏幕的仪表板。 AEM Screens的内容在渠道中进行 *管理*。 *AEM Screens播放器将渠道中显示的内容呈现到显示屏上* 。



>[!NOTE]
>
>要详细了解AEM Screens项目开发和管理中的不同功能，请参阅 **[AEM Screens用户指南](https://helpx.adobe.com/experience-manager/6-5/screens/user-guide.html)**。

## AEM Sites与AEM Screens {#aem-sites-screens}

> [!NOTE]
>
> 如果您的实施团队有使用AEM Sites应用程序的经验，请务必了解AEM Sites与AEM Screens之间的区别。

AEM Screens提供统一的创作／播放平台，用于将内容部署到公共空间中的数字标牌设备。 尽管该体验作者应努力保持跨Web和场内渠道的一致性，但还有一些不同之处需要注意。

* **停留时间**:通常，网页旨在提供大量可在相对更长的时间段内使用的信息。 相反，现场数字体验应预测观众的需求，并为用户应如何和为何参与提供清晰而简明的指导。 这样，体验就更具针对性、更精选性和情境性。

* **查看距离**:观看距离通常比用户在网站上体验的典型观看距离更长或更远。 因此，文本大小通常应该更大，并且应根据预期的屏幕大小和物理空间中的位置来测试文本、图像和其他免费内容之间的间距。

* **持久性**:播放器设备的连接状态不应允许影响是否将数字体验交付给用户。 播放器必须设计为始终保持一个或多个体验，并且无论播放器设备的连接状态如何，都可以提供。

* **媒体循环细分**:将每个播放器设备配置为具有其自己的循环区段可确保在整体数字体验中轻松创作、发布和播放本地化内容。 嵌入式序列频道中包含的媒体资产被添加到前一循环区段，并提供针对每个位置分组的媒体循环区段的机会。

* **交互式体验**:可使用AEM和SPA编辑器在Screens渠道中创作和交付触屏支持的自助服务终端应用程序。 最好应用一致的全渠道设计属性，不活动定时器来重置体验，并明确要求应用程序执行哪些任务。 登录页面应由关键数字元素组成，这些元素旨在传递价值、吸引用户进入屏幕并提示用户参与。

AEM Screens提供了一个框架，用于将内容部署到物理设备。 内容会分配给Screens中的渠道，这些渠道可以包含媒体内容或触摸屏应用程序。 在此框架中，AEM Site应用程序可以作为内容通过渠道交付。

将AEM站点放入屏幕中的渠道之前，必须对AEM站点进行格式化，以便在其要使用的显示设备尺寸上使用。

> [!NOTE]
>
> 许多AEM Sites组件与AEM Screens不兼容。 AEM Screens附带许多自带的现成组件，使您无需自定义即可构建数字体验。 如果项目要求允许，请尽可能使用内置AEM Screens功能。
